About the data

Where does the data come from?

All data is fetched in real time from the Balancing Mechanism Reporting Service (BMRS), operated by Elexon as part of the Balancing and Settlement Code (BSC) for the GB electricity system. BMRS is publicly available and requires no account or API key.

Which dataset is used?

The visualiser uses the B1610 — Actual Generation Output Per Generation Unit dataset (endpoint documentation). This reports the actual metered volume of electricity delivered to the grid by each registered Balancing Mechanism Unit (BMU) in every settlement period.

B1610 is the correct dataset for measuring what a generator actually exported, as opposed to what it planned to generate (Physical Notifications) or what it was instructed to adjust (Bid-Offer Acceptances).

What is Moray West?

Moray West is an offshore windfarm located in the Moray Firth, off the north-east coast of Scotland. It is registered in BMRS under the lead party Moray Offshore Wind West Ltd and consists of four BMUs:

The total farm output is the sum of all four BMUs. The farm became operational in 2024, so data before mid-2024 will be sparse or absent.

Note: The adjacent Moray East windfarm (BMU IDs T_MOWEO-1 to T_MOWEO-3) is a separate asset and is not included in this visualiser.

Key terms

BMU (Balancing Mechanism Unit)
A registered unit in the GB Balancing Mechanism. Each BMU represents a distinct block of generation or demand capacity. Moray West has four BMUs. You can search for any BMU at the BMRS BMU View.
Settlement Period
The GB electricity market operates in half-hour blocks called settlement periods. There are 48 per day (occasionally 46 or 50 on clock-change days). Each settlement period is numbered 1 to 48.
MWh (Megawatt-hour)
A unit of energy — the total amount of electricity produced or consumed over a period of time. The B1610 dataset reports values in MWh per settlement period.
MW (Megawatt)
A unit of power — the rate at which energy is being produced at any given moment. This visualiser converts MWh to average MW so that output can be compared against capacity benchmarks.
MWh to MW conversion
Per settlement period: Average MW = MWh ÷ 0.5 (since each period is half an hour).
Daily average: Average MW = total daily MWh ÷ 24.
Weekly average: Average MW = total weekly MWh ÷ (number of hours in the week).
B1610
The BMRS dataset identifier for "Actual Generation Output Per Generation Unit". It contains the metered energy output for every registered BMU. Endpoint documentation.
BMRS (Balancing Mechanism Reporting Service)
The platform operated by Elexon that publishes operational data about the GB electricity system. Available at bmrs.elexon.co.uk.
Elexon / BSC
Elexon administers the Balancing and Settlement Code (BSC), which governs the rules for balancing electricity supply and demand in Great Britain and settling the associated costs. elexon.co.uk.

How to use the visualiser

1. Select a date range

Use the From and To date pickers to set the period you want to view. The defaults are set to the last 30 days (offset by 5 days to account for data publication lag). Moray West data is available from mid-2024 onwards.

2. Choose which BMUs to include

The four BMU checkboxes let you toggle individual units on or off. When all four are selected, the chart shows the total farm output as a stacked bar. Deselecting a BMU removes its contribution from the chart, which is useful for comparing individual unit performance.

3. Set the aggregation level

The Aggregation dropdown controls how the data is grouped:

  • Per Settlement Period (30 min) — one bar per half-hour interval. Best for viewing the generation profile of a single day or a few days. MWh is divided by 0.5 to give average MW.
  • Daily Average — one bar per day. Total daily MWh is divided by 24 to give the daily average power in MW. Good for viewing weeks or months at a glance.
  • Weekly Average — one bar per week (grouped by week commencing Monday). Useful for longer-term trends over several months.

4. Fetch the data

Click Fetch & Visualise to call the BMRS API and render the chart. The status bar below the chart will show progress and the number of records returned.

5. Add reference lines

Use the Reference Lines panel to add horizontal lines at any MW value. For each line you can set:

  • Label — a description (e.g. "Installed Capacity", "90% Contractual Capacity")
  • MW value — the power level for the line (e.g. 882, 315)
  • Colour — click the colour swatch to pick any colour

Click Add Line to add it. Lines appear immediately on the chart as dashed horizontal lines. Click the × on any line's tag to remove it.

A default line at 315 MW (90% of contractual capacity) is included as a starting point but can be removed.

6. Read the chart

The chart is a stacked bar chart — each colour represents one BMU, and the total bar height is the combined output. Hover over any bar to see a tooltip with the per-BMU and total MW values for that period.

Below the chart, a summary line shows:

  • Total energy — the sum of all MWh across the selected period and BMUs
  • Peak avg MW — the highest bar value in the chart
  • Mean avg MW — the average of all bar values
  • Records — the number of raw data records returned by the API

Data caveats

  • ~5 day lag: B1610 data is published approximately five days after the operational period. It is not suitable for near-real-time monitoring.
  • Settlement run updates: Values may change slightly as later settlement runs replace the initial data. The most recent data reflects the latest available settlement run.
  • Raw units are MWh, not MW: The API returns energy (MWh) per settlement period. This visualiser converts to average MW for display. See the glossary above for the conversion method.
